git

git安装

window安装 http://git-scm.com/download/win下载Git客户端软件，和普通软件安装方式一样。
Linux 安装
CentOS发行版：sudo yum install git
Ubuntu发行版：sudo apt-get install git
Mac安装
打开Terminal直接输入git命令，会自动提示，按提示引导安装即可

git 工作原理

 git管理我们文件的三种状态，分别是已递交（committted),已修改(modified),已缓存（staged)由此引入git项目的桑工作区域概念：git仓库，工作目录，暂存区域
 基本的Git工作流程如下：
1、在工作目录中修改文件。
2、暂存文件，将文件的快照放入暂存区域。
3、提交文件，找到暂存区域的文件，将快照永久性存储到Git仓库目录。

配置用户

git config [option]
--global 全局配置，再当前用户下的所有仓库都共享同一个用户的配置信息
--system 系统配置，再该电脑下的所有仓库都共享同一个用户的配置信息
--local 本地配置,再该仓库下使用当前配置的用户信息
git config --global user.name "自已的名字"
git config --global user.email "自已的邮箱地址"
查看： git config --get user.name|git config --get user.email
替换：想要修改或者替换掉原先的用户信息，可以从新配置来实现。

初始化仓库

1 git init 只是创建了一个名为.git的隐藏目录。这个目录就是存储我们历史版本的仓库，ls -al可以查看
2 git clone 克隆仓库地址
3 git status 可以检测当前仓库文件的状态（会忽略孔目录）
4 git add 文件名 *或-A代表所有
 git add '所要添加的文件名' 添加某一个文件
 5 git commit -m "当前版本的描述信息"
 递交单个文件版本库，形成一个新版本：git commit filename -m "版本信息"
 6 git log 查看递交的历史版本 
 git reflog  查看所有历史递交版本
 7 git checkout filename 撤销指定文件修改
 再撤销更改的时候，如果暂存区 有该文件的话，就恢复暂存区的文件到工作目录。如果暂存区没有该文件，就从版本库中恢复。
 8 git reset [option] 回退版本
 git reset --hard '版本id-sha值' 恢复上一次递交的状态
 git reset  HEAD^ 回退到上一次递交的版本
 git reset HEAD^^ 回退到上上个版本
 git reset HEAD~2 回退到前面两个版本

git 分支

再初始化一个孔的本地仓库时，默认时没有主分支（master分支）的，必须在主分支上递交一次版本，git才会创建master分支。
git在创建分支时，必需保证当前仓库有master分支
1 创建分支 git branch name
 git checkout -b branchname 创建并切换到新分支
2 查看分支 git branch
有*标记的分支，表示当前工作目录所处的分支。
3 切换分支 git checkout name
4 合并分支 git merge branchname
将指定的分支合并到当前分支
5 删除分支 git branch -d branchname

git分支策略

在时机开发中，虽然git分支很强大，但是不能肆意使用，通常按照一定策略使用。提高开发效率
1 要保证主分支的稳定性，就是数保证主分支功能完整，且务bug。不能在主分支上开发
2 如果开发新功能，就穿概念一个开发该功能的分支，在该分支上惊醒编写。当功能开发完毕，并且测试务bug，将该分支合并到主分支上
该分支命名一半规范为:'dev' +相关功能描述的单词
3 如果在开发时原道协同开发，就要创建协同开发分支，在该分支上去编写代码。
